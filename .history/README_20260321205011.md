# US Healthcare EDI Parser and Validator (837 / 835 / 834)

A modern web-based platform to parse, validate, analyze, and visualize Healthcare EDI X12 files (837 Claims, 835 Remittance, 834 Enrollment). Built for speed, accuracy, and usability with AI-powered insights.

## Overview
Healthcare systems in the US rely heavily on EDI X12 transactions, but these files are hard to read, complex to validate, error-prone, and difficult to debug. This project solves that with:
- Human-readable EDI visualization
- Real-time validation and error detection
- AI-powered summaries
- Batch processing for multiple EDI files
- Claims vs payments reconciliation

## Supported EDI Transactions
| Transaction | Description |
| --- | --- |
| 837P | Professional healthcare claims |
| 837I | Institutional healthcare claims |
| 835 | Claim payment and remittance advice |
| 834 | Member enrollment and insurance data |
| 845 (optional) | Price authorization and purchasing |

## Key Features
### 1) EDI Parsing Engine
- Converts raw EDI into structured JSON
- Supports loops, segments, and elements
- Displays hierarchical structure (ISA to GS to ST to loops)

### 2) Real-Time Validation
- Syntax validation against X12 standards
- Segment-level validation
- Missing/invalid field detection
- Error highlighting with explanations

### 3) Batch Processing
- Upload a ZIP file containing multiple EDI files
- Auto-detect file types (837, 835, 834)
- Generate a consolidated validation report

### 4) 835 to 837 Reconciliation
- Match claims (837) with payments (835)
- Detect underpayments, denials, and missing claims
- Financial comparison dashboard

### 5) AI-Powered Assistant (Local LLM via Ollama)
- Explains EDI segments in plain English
- Summarizes claims and remittance
- Suggests fixes for validation errors
- Healthcare terminology explanation

### 6) Visualization Dashboard
- Claims distribution
- Payment trends
- Enrollment analytics
- Error frequency charts

### 7) Smart Insights
- Top denial reasons
- Most common validation errors
- Claim success rate
- Financial summaries

## Installation and Setup
### 1) Clone the Repository
```bash
git clone https://github.com/your-username/edi-parser.git
cd edi-parser
```

### 2) Backend Setup
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```

### 3) Frontend Setup
```bash
cd frontend
npm install
npm run dev
```

### 4) Setup Ollama (AI Assistant)
```bash
ollama run mistral
```

### 5) MongoDB Setup
- Install MongoDB locally or use Atlas
- Update the connection string in .env
