# 🏥 US Healthcare EDI Parser & Validator (837 / 835 / 834)

A modern web-based platform to **parse, validate, analyze, and visualize Healthcare EDI X12 files** (837 Claims, 835 Remittance, 834 Enrollment). Built for speed, accuracy, and usability with AI-powered insights.

---

## 🚀 Overview

Healthcare systems in the US rely heavily on **EDI X12 transactions**, but these files are:

* Hard to read
* Complex to validate
* Error-prone
* Difficult to debug

This project solves that by providing:

✅ Human-readable EDI visualization
✅ Real-time validation & error detection
✅ AI-powered summaries
✅ Batch processing of multiple EDI files
✅ Claims vs payments reconciliation

---

## 🎯 Supported EDI Transactions

| Transaction          | Description                        |
| -------------------- | ---------------------------------- |
| **837P**             | Professional healthcare claims     |
| **837I**             | Institutional healthcare claims    |
| **835**              | Claim payment & remittance advice  |
| **834**              | Member enrollment & insurance data |


---

## ✨ Key Features

### 🔍 1. EDI Parsing Engine

* Converts raw EDI into structured JSON
* Supports loops, segments, and elements
* Displays hierarchical structure (ISA → GS → ST → loops)

---

### ✅ 2. Real-Time Validation

* Syntax validation (X12 standards)
* Segment-level validation
* Missing/invalid field detection
* Error highlighting with explanations

---

### 📦 3. Batch Processing

* Upload ZIP file containing multiple EDI files
* Auto-detect file types (837, 835, 834)
* Generate consolidated validation report

---

### 🔄 4. 835 ↔ 837 Reconciliation

* Match claims (837) with payments (835)
* Detect:

	* Underpayments
	* Denials
	* Missing claims
* Financial comparison dashboard

---

### 🤖 5. AI-Powered Assistant (Local LLM via Ollama)

* Explains EDI segments in plain English
* Summarizes claims & remittance
* Suggests fixes for validation errors
* Healthcare terminology explanation

---

### 📊 6. Visualization Dashboard

* Claims distribution
* Payment trends
* Enrollment analytics
* Error frequency charts

---

### 🧠 7. Smart Insights

* Top denial reasons
* Most common validation errors
* Claim success rate
* Financial summaries

---

## 🛠️ Tech Stack

### Backend

* **Python (FastAPI)** – High-performance API framework
* **EDI Parsing** – Custom parser (or pyx12 alternative if needed)
* **File Handling** – `zipfile`, `pandas` for batch processing

---

### AI Layer

* **Local LLM (Ollama)**

	* Fast, private, no API cost
	* Used for:

		* EDI explanation
		* Error interpretation
		* Summarization

---

### Database

* **MongoDB**

	* Flexible schema for EDI JSON
	* Scalable for large healthcare datasets

---

### Frontend

* **React.js**
* **Tailwind CSS**
* **Charts: Recharts / Chart.js**
